# SBTI Design System

## Design Context

### Users
- **Who**: 年轻互联网用户（18-30岁），熟悉 meme 文化，活跃于社交媒体
- **Context**: 闲暇时间刷手机、朋友间分享娱乐内容、社交场景中需要话题
- **Job to be Done**: 用一个好玩的测试获得有趣的自我标签，截图分享到社交媒体引发讨论
- **Emotional Goals**: 
  - 轻松解压 — 无压力的娱乐体验，不需要认真对待
  - 好笑到想分享 — 结果足够有趣/精准，驱动用户主动传播
  - 社交破冰 — 朋友互测互比，促进对话和讨论

### Brand Personality
- **3 Words**: 荒诞 × 潮流 × 社交
- **Voice**: 自嘲式幽默，不说教、不装严肃，像一个很会讲段子的损友
- **Tone**: 夸张但不恶意，犀利但不刻薄，自我解构式的讽刺
- **Emotional Register**: 让人笑出声 → 觉得被说中 → 想给朋友看

### Aesthetic Direction
- **Visual Tone**: 大胆、色彩丰富、视觉冲击力强，但不廉价
- **Positive References**: 
  - BuzzFeed 风格测试 — 活泼、色彩丰富、社交化，但需要更高级的执行质感
  - 16personalities.com — 清晰的信息架构和结果展示，但需要更大胆和有趣
- **Anti-references**: 
  - 不要传统心理学测试的严肃感和学术气质
  - 不要过于极简/冷淡的科技风（Linear/Vercel 风格太严肃）
  - 不要看起来像廉价的 H5 小游戏
- **Theme**: 系统跟随（light/dark），两种模式都需要出色的视觉表现
- **Key Visual Elements**:
  - 大胆的排版和字体层级
  - 丰富但协调的色彩运用（不是单调灰度）
  - 结果页面要足够"出片"，适合截图分享
  - 人格类型的视觉识别度要强

### Design Principles

1. **Shareable First** — 每个页面状态（尤其是结果页）都要考虑"截图分享到社交媒体后好不好看"。视觉设计服务于传播。

2. **Bold but Refined** — 大胆≠粗糙。色彩要丰富但协调，排版要有冲击力但有层次，荒诞的内容需要精致的容器来承载。

3. **Instant Delight** — 用户不需要学习如何使用。每一步都应该自然流畅，带有小惊喜（微动效、文案彩蛋、视觉细节）。

4. **Social by Design** — 设计应该鼓励对话和比较。人格类型要有强视觉识别度，让人一看截图就想"我也要测"。

5. **Humor in Every Pixel** — 不仅文案要好笑，视觉语言也应该传递幽默感。设计本身可以是段子的一部分。

### Technical Constraints
- **Stack**: Next.js 16 + Tailwind CSS v4 + shadcn/ui (base-nova) + Base-UI
- **Deployment**: Cloudflare Pages (static export)
- **i18n**: 4 languages (zh, en, ja, ko) via next-intl
- **Color System**: OKLch color space
- **Current Issue**: 当前设计过于单调（几乎全灰度），缺少品牌色和视觉活力，与"荒诞×潮流×社交"的定位不匹配
- **Images**: 27 personality type images already exist in /public/types/
- **Accessibility**: Base-UI provides ARIA/keyboard/screen reader support; maintain WCAG AA contrast ratios
